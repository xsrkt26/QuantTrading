# 量化交易学习 RoadMap

这个 RoadMap 是本仓库的长期学习主线。目标不是寻找暴富捷径，而是建立一套可复现、可验证、可迭代、可控风险的量化研究和交易能力。

## 核心原则

- 先研究，后交易；先模拟，后小资金；先风控，后收益。
- 每个策略必须有明确假设，不能只看收益曲线。
- 每个回测必须记录数据来源、样本区间、手续费、滑点、参数和限制。
- 没有样本外验证的策略，只能算想法，不能算策略。
- 不追求一次暴利，追求长期可复制的研究流程。
- 不把 AI 当答案机，要把 AI 当导师、助教、代码陪练和审稿人。
- 核心市场观记录在 `notes/2026-05-19-core-quant-principles.md`，每做一个策略都要用其中的检查清单审一遍。

## 最终能力目标

完成本路线后，应能独立完成以下工作：

- 获取、清洗、校验市场数据。
- 编写可复用的收益、风险、回撤、绩效指标。
- 从策略假设出发完成回测、复盘和报告。
- 识别未来函数、过拟合、幸存者偏差、数据泄露等常见陷阱。
- 研究基础因子并评估其稳定性。
- 构建组合、控制仓位和风险暴露。
- 将研究代码逐步工程化，为模拟盘和小资金实盘做准备。

## 阶段 0：学习系统和仓库规范

目标：把学习过程变成可积累的系统，而不是散乱笔记。

学习内容：

- Git 基础：提交、分支、远程推送。
- 项目结构：`notes/`、`research/`、`strategies/`、`backtests/`、`src/`、`reports/`。
- 复盘方法：每次实验记录问题、过程、结果、下一步。

产出：

- `README.md`：项目入口。
- `ROADMAP.md`：学习路线图。
- `resources/books.md`：经典书单。
- `notes/2026-05-19-core-quant-principles.md`：量化核心观念和策略检查清单。
- 每周至少一次 Git 提交。

验收标准：

- 任意一个研究或策略，都能从仓库中找到对应笔记、代码和结论。

## 每节课交付规范

每节课默认包含四类文件：

- `notebooks/NN_topic.ipynb`：可运行代码讲义。
- `notes/YYYY-MM-DD-topic.md`：概念笔记。
- `notes/YYYY-MM-DD-topic-explained.md`：逐段讲解，用来配合 notebook 阅读。
- `reports/YYYY-MM-DD_topic.md`：有实证结果时保存正式报告。

以后每次新增讲义后，都要补一份 `*-explained.md`，说明代码每一段在做什么、为什么这么做、结果应该如何解读。

## 阶段 1：Python 数据分析基础

目标：能熟练处理日线行情数据，并画出基础分析图。

学习内容：

- Python 基础：函数、类、模块、虚拟环境、包管理。
- `numpy`：数组、向量化计算、常用统计函数。
- `pandas`：DataFrame、时间索引、缺失值、滚动窗口、分组。
- 可视化：净值曲线、收益分布、回撤曲线、K 线。

练习任务：

- 读取一份股票、ETF 或指数日线数据。
- 计算日收益率、累计收益率、滚动均线。
- 绘制价格曲线、净值曲线和回撤曲线。

产出：

- `notebooks/01_market_data_basics.ipynb`
- `notes/YYYY-MM-DD-market-data-basics.md`

验收标准：

- 能解释每一列数据的含义。
- 能说明为什么要处理缺失值、停牌、复权和异常价格。

## 阶段 2：收益、风险和绩效指标

目标：不依赖黑箱工具，自己写出常用策略评价指标。

学习内容：

- 简单收益率、对数收益率、累计收益。
- 年化收益、年化波动率、夏普比率、Calmar 比率。
- 最大回撤、回撤持续时间、胜率、盈亏比。
- 换手率、交易成本、滑点、容量。

练习任务：

- 手写最大回撤函数。
- 手写夏普比率和 Calmar 比率。
- 对一组随机收益和一组真实收益做对比。

产出：

- `src/metrics/returns.py`
- `src/metrics/risk.py`
- `tests/test_metrics.py`
- `notes/YYYY-MM-DD-performance-metrics.md`

验收标准：

- 指标函数有测试。
- 能说清每个指标适用场景和缺陷。

## 阶段 3：第一个完整回测

目标：从策略假设到回测报告形成闭环。

策略候选：

- 双均线策略。
- 动量策略。
- 均值回归策略。
- 突破策略。

学习内容：

- 交易信号生成。
- 信号滞后和未来函数检查。
- 仓位、现金、持仓、市值、手续费。
- 样本内和样本外拆分。
- 参数敏感性分析。

练习任务：

- 选择一个简单策略。
- 写出策略假设。
- 完成向量化回测。
- 加入手续费和滑点。
- 输出回测图表和指标表。

产出：

- `strategies/first_strategy/README.md`
- `backtests/YYYY-MM-DD_first_strategy.md`
- `notebooks/02_first_backtest.ipynb`

验收标准：

- 策略规则清晰到别人可以复现。
- 回测中没有使用未来数据。
- 报告中必须写失败原因和改进方向。

## 阶段 4：经典策略族

目标：理解不同策略赚钱逻辑，而不是堆指标。

学习内容：

- 趋势跟踪：均线、突破、时间序列动量。
- 横截面动量：排名、调仓、组合持仓。
- 均值回归：价格偏离、价差回归、配对交易。
- 波动率策略：波动率过滤、风险目标、仓位缩放。
- 事件驱动入门：财报、公告、指数调整。

练习任务：

- 每类策略至少做一个最小可运行版本。
- 比较不同策略在震荡、趋势、暴跌环境下的表现。
- 记录策略适用市场环境和失效条件。

产出：

- `strategies/trend_following/README.md`
- `strategies/momentum/README.md`
- `strategies/mean_reversion/README.md`
- `reports/strategy-family-review.md`

验收标准：

- 能解释策略收益来自趋势、风险溢价、行为偏差还是市场结构。
- 能指出每类策略最怕什么市场环境。

## 阶段 5：因子研究

目标：学会评估一个因子是否有稳定解释力。

学习内容：

- 因子构造：价值、动量、质量、低波、规模、成交量。
- 因子预处理：去极值、标准化、中性化、缺失值处理。
- 因子检验：分层回测、IC、Rank IC、换手率。
- 稳健性：不同市场、不同周期、不同参数、不同交易成本。
- 多因子组合：等权、IC 加权、风险约束。

练习任务：

- 构造至少 3 个简单因子。
- 做分层回测。
- 计算 IC 和 Rank IC。
- 写出因子是否值得继续研究的结论。

产出：

- `research/factors/README.md`
- `research/factors/factor_template.md`
- `notebooks/03_factor_research.ipynb`

验收标准：

- 因子结论不能只看单次收益。
- 必须检查稳定性、换手率和交易成本。

## 阶段 6：组合构建和仓位管理

目标：从“买什么”升级到“买多少”和“承担多少风险”。

学习内容：

- 等权组合。
- 波动率倒数加权。
- 风险平价。
- 最大仓位、行业约束、单票约束。
- 再平衡频率。
- 风险预算和回撤控制。

练习任务：

- 对多个策略或多个资产构建组合。
- 比较等权、波动率倒数、风险平价。
- 加入最大回撤控制规则。

产出：

- `src/portfolio/`
- `notebooks/04_portfolio_construction.ipynb`
- `reports/portfolio-construction-review.md`

验收标准：

- 能解释组合收益、风险和相关性的关系。
- 能说明为什么单个高收益策略不一定适合重仓。

## 阶段 7：金融统计和时间序列

目标：建立足够的统计直觉，避免把噪声当规律。

学习内容：

- 随机变量、分布、置信区间、假设检验。
- 自相关、偏自相关、平稳性。
- 回归、残差、异方差、多重共线性。
- ARIMA、GARCH、协整、配对交易。
- 多重检验和数据挖掘偏差。

练习任务：

- 检查收益率自相关。
- 对一个配对交易样本做协整检验。
- 比较样本内显著和样本外失效案例。

产出：

- `notes/YYYY-MM-DD-financial-time-series.md`
- `notebooks/05_time_series_basics.ipynb`

验收标准：

- 能解释“统计显著”不等于“可交易”。
- 能说明为什么样本外验证很重要。

## 阶段 8：机器学习量化

目标：知道机器学习在量化中何时有用，何时只是过拟合。

学习内容：

- 特征工程：价格、成交量、波动率、基本面、另类数据。
- 标签设计：未来收益、分类标签、三重障碍法。
- 训练验证：时间序列切分、交叉验证、样本外测试。
- 模型：线性模型、树模型、随机森林、梯度提升。
- 风险：数据泄露、标签泄露、过拟合、特征选择偏差。

练习任务：

- 用简单机器学习模型预测未来收益方向。
- 与朴素动量或均值回归基准比较。
- 写出模型为什么可能失效。

产出：

- `research/ml/README.md`
- `notebooks/06_ml_signal_baseline.ipynb`

验收标准：

- 模型必须和简单基准比较。
- 不能只报告准确率，必须报告交易表现和成本影响。

## 阶段 9：工程化和自动化

目标：让研究、回测和报告可复现。

学习内容：

- 项目配置：参数文件、路径管理、日志。
- 数据管线：下载、清洗、缓存、校验。
- 回测框架：向量化回测、事件驱动回测、组合层回测。
- 自动报告：指标表、图表、交易明细、参数表。
- 测试：单元测试、回归测试、数据质量测试。

练习任务：

- 把 notebook 中稳定逻辑迁移到 `src/`。
- 为指标函数和回测核心逻辑写测试。
- 自动生成一次回测报告。

产出：

- `src/data/`
- `src/backtest/`
- `src/reporting/`
- `tests/`

验收标准：

- 换一台机器后，能根据 README 复现核心结果。
- 关键函数有测试，重要参数有记录。

## 阶段 10：模拟盘和实盘前准备

目标：理解实盘和回测之间的差异，不在没有准备时冒险。

学习内容：

- 行情延迟、成交延迟、订单状态。
- 手续费、滑点、冲击成本、容量。
- 异常处理：断网、停牌、涨跌停、数据缺失。
- 风控：最大仓位、最大亏损、最大回撤、黑名单。
- 监控：日志、告警、持仓核对、成交核对。

实盘前禁止事项：

- 没有完成回测报告，不实盘。
- 没有样本外测试，不实盘。
- 没有交易日志和风控规则，不实盘。
- 没有模拟盘记录，不加大资金。
- 不用借贷资金，不用不可承受亏损的资金。

产出：

- `reports/live-trading-checklist.md`
- `reports/paper-trading-log.md`
- `reports/risk-policy.md`

验收标准：

- 任何一次交易都能解释为什么买、为什么卖、亏损多少必须停止。

## 每次策略研究模板

每个策略至少回答以下问题：

- 策略假设是什么？
- 这个假设为什么可能赚钱？
- 用了哪些数据？
- 样本区间是什么？
- 是否有未来函数？
- 手续费和滑点如何设置？
- 样本内表现如何？
- 样本外表现如何？
- 参数是否敏感？
- 最大回撤来自什么时期？
- 什么情况下策略应该停用？
- 下一步是继续、修改还是放弃？

## 推荐阅读顺序

读书用于校准框架，不用于替代实践。

1. `Inside the Black Box`：量化投资全貌。
2. `Quantitative Trading`：个人量化交易入门。
3. `Algorithmic Trading`：策略开发和实战。
4. `Expected Returns`：风险溢价和资产收益来源。
5. `Active Portfolio Management`：主动组合管理经典框架。
6. `Analysis of Financial Time Series`：金融时间序列基础。
7. `Advances in Financial Machine Learning`：机器学习量化和防过拟合。
8. `Trading and Exchanges`：市场微观结构。
