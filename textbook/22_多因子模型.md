## 讲解

- 多因子组合要检查因子之间是否高度重复。
- 单因子强弱会轮换，组合因子的目标是降低单一来源失效风险。
- 如果 composite 只是复制 momentum，那就不是真正的多因子。

## 详细讲解

### 1. 第 22 章在第 19-21 章之后解决什么问题

第 19 章讲了：

```text
单个因子如何打分、排序、分层回测。
```

第 20 章讲了：

```text
如何用 Rank IC 和换手率检查因子稳定性。
```

第 21 章讲了：

```text
如何检查因子是不是混入了 beta 或波动率暴露。
```

第 22 章开始问一个更实际的问题：

```text
如果单个因子都不够强，多个弱因子能不能组合成更稳的信号？
```

这就是多因子模型的入口。

### 2. 多因子不是堆指标

很多初学者会以为：

```text
指标越多，模型越高级。
```

这是危险的。

多因子的重点不是数量，而是：

```text
每个因子是否有独立解释力？
这些因子是否真的代表不同收益来源？
合成后是否比单个因子更稳？
```

如果你把 10 个高度相似的动量指标加在一起，本质上还是一个动量因子，只是看起来更复杂。

### 3. 本章用了哪三个因子

本章用了 3 个 ETF 横截面因子：

```text
momentum：中期动量
reversal：短期反转
low_volatility：低波动
```

它们分别代表三种不同想法。

`momentum` 问的是：

```text
过去 6 个月涨得更好的 ETF，未来是否继续更强？
```

`reversal` 问的是：

```text
最近 1 个月跌得多的 ETF，未来是否有短期反弹？
```

`low_volatility` 问的是：

```text
波动更低的 ETF，未来是否有更好的风险调整表现？
```

这三个因子不应该是完全一样的，否则组合意义不大。

### 4. 代码里三个因子怎么构造

实现里核心逻辑是：

```python
momentum = monthly / monthly.shift(6) - 1
reversal = -(monthly / monthly.shift(1) - 1)
low_vol = -(daily_returns.rolling(63).std() * np.sqrt(252)).resample("ME").last()
```

`momentum` 是过去 6 个月收益，越高越好。

`reversal` 前面有一个负号。意思是：

```text
最近 1 个月跌得越多，reversal 分数越高。
```

因为短期反转假设认为短期过度下跌后可能反弹。

`low_vol` 也有一个负号。意思是：

```text
波动率越低，low_volatility 分数越高。
```

因为原始波动率越高代表风险越高，而低波动因子希望偏向低风险资产。

### 5. 为什么要做 z-score 标准化

三个因子的原始量纲不同：

```text
momentum 是 6 个月收益率
reversal 是 1 个月反转收益
low_volatility 是波动率
```

如果直接相加，数值尺度大的因子会天然占优势。

所以本章先做横截面标准化：

```python
z_momentum = zscore(momentum)
z_reversal = zscore(reversal)
z_low_volatility = zscore(low_volatility)
```

标准化后，每个月每个因子都变成相对分数：

```text
高于当月资产池平均值 -> 正分
低于当月资产池平均值 -> 负分
```

这样三个因子才有资格相加。

### 6. composite 是什么

课本里的核心代码是：

```python
score = z_momentum + z_reversal + z_low_volatility
portfolio = score.rank(axis=1, pct=True)
```

实现里是：

```python
composite = sum(_zscore_cross_sectional(factor) for factor in factors.values()) / len(factors)
```

意思是：

```text
把三个标准化后的因子等权平均，得到综合分数。
```

如果某个 ETF：

```text
中期动量强
短期没有过热
波动率较低
```

它的 composite 分数就可能较高。

### 7. 本章的收益是怎么测的

本章仍然使用因子研究的多空框架：

```text
每个月按因子分数排序；
做多最高 20%；
做空最低 20%；
观察 high_minus_low 收益。
```

所以表里的 `momentum`、`reversal`、`low_volatility`、`composite` 不是普通只做多组合，而是：

```text
对应因子的高分组 - 低分组。
```

这和第 19 章的 `high_minus_low` 是同一类研究组合。

它的作用是看因子本身有没有相对收益，不是直接给你一个完整实盘策略。

### 8. 用 100W 账户怎么理解

如果用 `composite` 做研究组合，可以粗略理解成：

```text
100W 做多综合分最高的一组 ETF；
同时做空综合分最低的一组 ETF。
```

如果只做 long-only 简化版本，则可以理解成：

```text
每个月买 composite 分数最高的约 20% ETF；
这些 ETF 之间等权；
下个月重新打分和换仓。
```

例如某个月最高分组是：

```text
XLK, QQQ, XLU
```

那么 long-only 账户可能是：

```text
XLK：33.3W
QQQ：33.3W
XLU：33.3W
```

但本章表格里的结果不是这个 long-only 版本，而是高低分组相减的因子收益。

### 9. 如何读本章结果

本章结果是：

```text
momentum：ann_return 0.22%，max_drawdown -45.13%
reversal：ann_return -6.09%，max_drawdown -72.63%
low_volatility：ann_return -0.25%，max_drawdown -54.43%
composite：ann_return 1.10%，max_drawdown -41.33%
```

这说明：

```text
三个单因子都不强；
reversal 尤其差；
composite 比单因子略好，但仍然很弱。
```

所以第 22 章不能解读成“多因子模型成功了”。更准确的结论是：

```text
等权合成确实让结果比几个弱单因子稍微稳一点，但这些因子的整体质量还不够。
```

### 10. 为什么 composite 仍然有价值

虽然 `composite` 的结果不强，但它展示了一个重要研究流程：

```text
先构造多个因子；
统一尺度；
检查相关性；
合成总分；
比较单因子和综合因子。
```

这个流程非常重要。

以后如果我们换成更有信息的因子，或者换成更大的股票池，这套流程仍然适用。

本章真正训练的是研究框架，而不是证明这三个 ETF 因子已经能赚钱。

### 11. 因子相关性为什么重要

图里的右侧是 factor correlation，也就是因子之间的相关性。

如果两个因子高度相关，比如：

```text
momentum 和另一个 5 个月动量因子相关性很高
```

那它们组合在一起不会带来多少新信息。

真正有价值的多因子组合，应该尽量包含不同来源：

```text
趋势类
反转类
风险类
估值类
质量类
情绪类
```

当然，因子不同不代表一定有效。它们还必须各自有合理解释和统计证据。

### 12. 多因子最容易犯的错误

多因子最常见的错误是：

```text
看到一个因子历史差，就调低；
看到一个因子历史好，就调高；
不断调整权重直到回测好看。
```

这很容易过拟合。

更稳妥的做法是先定义原则：

```text
为什么这些因子应该长期有效？
它们是否代表不同风险或行为偏差？
它们的相关性是否足够低？
它们在样本外是否还能工作？
```

然后再谈权重。

### 13. 本章过关标准

你能讲清楚下面四句话，第 22 章就算过关：

```text
多因子不是堆指标，而是组合不同来源的有效信号。
不同因子需要先标准化，否则不能直接相加。
composite 是多个标准化因子的等权综合分。
本章 composite 只是略好于单因子，说明流程成立，但因子质量仍然偏弱。
```
