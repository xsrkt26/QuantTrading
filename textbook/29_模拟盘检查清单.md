## 讲解

- 模拟盘日志要记录信号、订单、成交、持仓和检查项。

## 详细讲解

### 1. 为什么回测通过后不能直接实盘

很多新手会以为：

```text
回测赚钱 -> 可以实盘。
```

这是危险的。

回测只证明了一件事：

```text
在你的历史数据和假设下，策略过去表现如何。
```

它还没有证明：

```text
每天信号能否稳定生成；
订单能否正确发出；
成交价格是否合理；
持仓是否和系统记录一致；
异常情况能否被发现；
日志能否解释每一次交易。
```

这些问题需要模拟盘来验证。

所以模拟盘不是可有可无的仪式，而是从研究代码到实盘系统之间的桥。

### 2. 模拟盘到底在模拟什么

模拟盘不是为了模拟你能赚多少钱。

它主要模拟的是流程：

```text
今天信号是什么；
根据当前持仓要不要下单；
如果下单，订单方向和数量是多少；
假设用什么价格成交；
成交后持仓是否正确；
当天收益和日志是否能对上。
```

本章关键代码是：

```python
paper_log.append({'date': today, 'signal': signal, 'order': order, 'fill': fill})
```

真正重要的不是这一行代码本身，而是它背后的观念：

```text
每一天都要留下可复盘记录。
```

没有日志，出了问题只能靠猜。

### 3. 本章模拟盘日志包含什么

本章用最近 80 个交易日做模拟盘流程演示。

日志里包含：

```text
signal：当天策略信号；
position：根据滞后信号形成的持仓；
order：今天需要调整的仓位；
fill_price：假设成交价格；
close：当天收盘价；
slippage_bps：开盘价相对前收盘的跳空幅度；
strategy_return：模拟策略收益；
signal_checked：信号是否检查；
order_checked：订单是否检查；
position_checked：持仓是否检查；
checklist_complete：检查清单是否完成。
```

这比只看净值曲线重要得多。

因为实盘后你需要回答：

```text
为什么今天买了？
为什么买这么多？
成交价格是什么？
买完后仓位对不对？
这笔交易是否符合策略？
```

### 4. 信号核对是什么

信号核对回答：

```text
策略今天到底想不想持仓？
```

你要检查：

```text
数据是否更新到最新交易日；
均线、因子、模型特征是否正常；
信号日期有没有错位；
今天的信号和昨天的信号相比是否发生变化；
信号变化是否符合策略逻辑。
```

很多实盘错误并不是模型判断错，而是：

```text
数据没更新；
信号用错日期；
复权数据和实时价格混用；
字段缺失但代码默认填 0。
```

这些错误必须在模拟盘阶段发现。

### 5. 订单核对是什么

订单核对回答：

```text
从当前持仓到目标持仓，需要下什么单？
```

比如：

```text
当前 position = 0；
目标 position = 1；
order = +1。
```

如果当前已经持有，目标也继续持有：

```text
当前 position = 1；
目标 position = 1；
order = 0。
```

订单核对可以防止：

```text
重复买入；
该卖没卖；
买卖方向反了；
仓位单位算错；
现金不足仍然下单。
```

实盘里这类错误比你想象的常见。

### 6. 持仓核对是什么

持仓核对回答：

```text
成交后，系统记录的持仓和券商账户是否一致？
```

回测里 position 是一列数字，很干净。

实盘里你会遇到：

```text
订单部分成交；
订单被拒；
成交后账户没及时刷新；
手续费和税费导致现金变化；
分红、拆股、合股影响数量；
多策略同时交易同一标的。
```

如果你不核对持仓，策略可能以为自己是空仓，实际上还持有；或者以为自己满仓，实际上订单没有成交。

这就是模拟盘要提前演练的东西。

### 7. 本章实跑结果怎么读

本章结果是：

```text
paper_days = 80
orders = 3
average_slippage_bps = 45.0539
checklist_completion_rate = 1.0000
paper_final_equity = 0.9898
```

这说明最近 80 个交易日里：

```text
模拟了 80 天；
发生了 3 次仓位变化；
检查清单完成率是 100%；
模拟净值略低于 1；
开盘价相对前收盘的平均跳空幅度约 45 bps。
```

这里最重要的不是 `paper_final_equity = 0.9898`。

模拟盘短期亏一点不代表策略坏了，短期赚很多也不代表策略可以实盘。

真正重要的是：

```text
信号、订单、成交、持仓、日志流程有没有跑通。
```

### 8. slippage_bps 怎么理解

本章里的 `average_slippage_bps` 用开盘价相对前一日收盘价的跳空幅度近似观察执行不确定性。

它不是完整实盘滑点。

真实滑点还会受影响：

```text
bid-ask spread；
订单大小；
成交量；
下单时机；
市场冲击；
券商路由；
盘前盘后跳空。
```

但它提醒你一件事：

```text
回测里的成交价很干净，实盘里的成交价会抖动。
```

尤其是低流动性股票，这个问题会更严重。

### 9. 模拟盘至少要跑多久

没有固定答案，但至少要覆盖：

```text
几次买入；
几次卖出；
一次信号保持不变；
一次数据异常处理；
一次无交易日或节假日；
一次手工复核流程。
```

如果一个低频策略一个月只交易一次，那么跑一周模拟盘没有意义。

模拟盘的长度要跟策略频率匹配。

对日线低频策略来说，比较合理的要求是：

```text
至少跑完若干次完整信号变化；
确保日志能解释每一天的状态；
确认你能稳定执行而不是靠临时手工补救。
```

### 10. 本章过关标准

你能讲清楚下面四句话，第 29 章就算过关：

```text
模拟盘的目标不是证明赚钱，而是验证研究到交易的流程。
每天必须记录信号、订单、成交、持仓和检查状态。
信号核对、订单核对、持仓核对分别解决不同类型的实盘错误。
没有可复盘日志，就不能把策略问题和执行问题分开。
```
