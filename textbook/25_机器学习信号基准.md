## 详细讲解

### 1. 第 25 章为什么要从基准模型开始

很多人一听机器学习量化，就会直接想到复杂模型：

```text
神经网络
随机森林
XGBoost
深度学习
强化学习
```

但正确的学习顺序不是先上复杂模型，而是先建立一个非常朴素的基准：

```text
简单模型能不能在严格测试集里超过简单规则？
```

如果一个线性基准模型都没有稳定价值，直接上复杂模型通常只会更容易过拟合。

### 2. 机器学习量化的基本结构

机器学习策略一般有三个核心部分：

```text
特征 features：今天已知的信息
标签 label：未来要预测的东西
模型 model：从特征到预测的函数
```

本章的任务是：

```text
用今天已经知道的 SPY 特征，预测下一天 SPY 收益方向或收益大小。
```

然后把预测值转成交易信号。

### 3. 本章用了哪些特征

实现里使用了这些特征：

```text
ret_5：过去 5 日收益
ret_20：过去 20 日收益
ret_63：过去 63 日收益
vol_20：过去 20 日年化波动率
volume_z：成交量相对过去 20 日均值的 z-score
```

这些都是在当天能看到的数据。

这点很关键：

```text
特征必须来自预测时点以前。
```

如果把未来收益、未来最高价、未来成交量放进特征，模型结果会非常好看，但那是数据泄露，不是策略能力。

### 4. 标签是什么

本章标签是下一天收益：

```python
future_1d_return = returns.shift(-1)
```

意思是：

```text
用今天的特征，预测明天的收益。
```

模型训练时拟合的是收益数值，不只是涨跌分类。

后面交易时再把预测值转成信号：

```python
signal = prediction > 0
```

也就是：

```text
预测明天收益为正 -> 持有 SPY
预测明天收益不为正 -> 空仓
```

### 5. 为什么要做时间切分

本章用时间顺序切分训练集和测试集：

```text
前 65% 数据：训练集
后 35% 数据：测试集
```

这是时间序列机器学习的基本纪律。

不能随机打乱训练和测试，因为金融数据有时间顺序。如果随机打乱，模型可能在训练时见过未来市场环境，测试结果会虚高。

正确做法是：

```text
只用过去训练；
只在未来测试。
```

### 6. Ridge 模型是什么

本章用的是 Ridge 线性模型。

你可以先把它理解成：

```text
带正则化的线性回归。
```

它会学习每个特征的权重：

```text
prediction = b0 + b1 * ret_5 + b2 * ret_20 + b3 * ret_63 + b4 * vol_20 + b5 * volume_z
```

正则化的作用是：

```text
限制参数不要过度放大，降低过拟合风险。
```

本章故意用简单模型，是为了建立机器学习基准，而不是追求模型炫技。

### 7. 标准化为什么只用训练集

实现里先用训练集计算均值和标准差：

```python
mean = x_train.mean()
std = x_train.std(ddof=0)
```

然后用同一组均值和标准差处理训练集、测试集。

这点很重要。

如果你用全样本计算均值和标准差，就等于让测试集信息参与了训练前处理，也是一种轻微的数据泄露。

机器学习量化里，任何预处理都必须遵守：

```text
只能从训练集学习参数；
再把这些参数应用到测试集。
```

### 8. 预测值如何变成交易

本章交易规则非常简单：

```text
prediction > 0：持有 SPY
prediction <= 0：空仓
```

如果账户是 100W：

```text
预测为正 -> 100W 买 SPY
预测为非正 -> 100W 持现金
```

它不是复杂仓位模型，也没有根据预测强弱调整仓位。

后续可以扩展成：

```text
预测值越高，仓位越高；
预测值接近 0，不交易；
预测值为负，空仓或做空。
```

但第 25 章先保持最小可理解版本。

### 9. 为什么要和 momentum_baseline 比较

机器学习模型不能只和 0 比，也不能只说自己赚钱。

它必须和简单规则比较。

本章的简单基准是：

```text
momentum_baseline：用 ret_20 作为预测分数。
```

也就是：

```text
过去 20 天涨 -> 看多
过去 20 天跌 -> 空仓
```

如果机器学习模型连这个简单规则都赢不了，那复杂模型就没有实际价值。

### 10. 如何读本章结果

本章测试集结果是：

```text
ridge_features final_equity = 2.3072
momentum_baseline final_equity = 2.2367
buy_hold_test final_equity = 2.7888
```

从最终净值看，Ridge 略高于简单动量基准，但低于测试期买入持有。

再看 Sharpe：

```text
ridge_features sharpe = 0.7688
momentum_baseline sharpe = 1.0514
buy_hold_test sharpe = 0.7935
```

简单动量基准的 Sharpe 反而最高。

再看最大回撤：

```text
ridge_features max_drawdown = -22.24%
momentum_baseline max_drawdown = -13.86%
buy_hold_test max_drawdown = -33.72%
```

Ridge 降低了买入持有的回撤，但没有简单动量基准稳。

所以正确结论是：

```text
Ridge 模型有一点效果，但没有明显战胜简单动量基准。
```

### 11. 方向准确率为什么不能单独看

本章结果：

```text
ridge_features direction_accuracy = 50.68%
momentum_baseline direction_accuracy = 53.05%
```

方向准确率只是：

```text
预测涨跌方向猜对了多少次。
```

但交易收益还取决于：

```text
猜对时赚多少；
猜错时亏多少；
什么时候持仓；
换手率和成本；
大跌时是否避开。
```

一个模型准确率高但亏钱，完全可能。一个模型准确率略低但盈亏比好，也可能赚钱。

所以机器学习策略不能只汇报 accuracy，必须看净值、回撤、Sharpe、成本和稳定性。

### 12. 本章还缺什么

第 25 章只是 baseline，还不是完整机器学习量化系统。

后续至少还要做：

```text
滚动训练和 walk-forward 验证
更严格的防泄露检查
特征稳定性分析
交易成本和滑点敏感性
不同市场阶段表现
预测值分层检验
仓位和风控规则
```

第 26 章会专门讲机器学习里最危险的问题：

```text
数据泄露。
```

### 13. 本章过关标准

你能讲清楚下面四句话，第 25 章就算过关：

```text
机器学习策略由特征、标签、模型和交易规则组成。
训练集必须早于测试集，不能随机打乱时间序列。
模型必须和简单基准比较，否则复杂度没有意义。
方向准确率不是最终目标，交易指标才是最终检验。
```
