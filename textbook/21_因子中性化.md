## 本课问题

因子有效是真因子有效，还是暴露了 beta 或波动率？

## 必须理解的概念

- 去极值
- 标准化
- beta 中性
- 波动率中性
- 残差化

## 讲解

- 中性化后 IC 变好，说明原因子里可能有不想要的 beta/波动率暴露。
- 中性化后 IC 变差，也不等于错误，可能说明暴露本身就是收益来源。
- 中性化要服务于问题定义，而不是机械套用。

## 详细讲解

### 1. 第 21 章为什么要讲中性化

第 19、20 章发现一个问题：

```text
这个 ETF 动量因子的排序能力不稳定。
```

第 21 章继续问：

```text
这个因子弱，是因为它本身没信息？
还是因为它混入了我们不想要的风险暴露？
```

比如一个动量因子看起来有效，可能不是因为它真的捕捉了 alpha，而是因为它偏向了：

```text
高 beta 资产
高波动资产
某些行业 ETF
某种市场风格
```

中性化就是用来诊断这个问题的。

### 2. beta 暴露是什么

`beta` 可以理解成：

```text
一个资产对市场涨跌的敏感度。
```

本章用 SPY 作为市场代表。

如果某个 ETF 的 beta 高，通常表示：

```text
SPY 涨时它更容易涨；
SPY 跌时它也更容易跌。
```

如果一个因子总是给高 beta 资产高分，那它可能不是在选真正的 alpha，而是在选“更激进的市场暴露”。

这种收益不一定错，但你要知道自己赚的是什么。

### 3. 波动率暴露是什么

波动率是价格波动程度。

本章用滚动 63 日波动率估计每个 ETF 的风险水平：

```python
volatility = daily_returns.rolling(63).std() * np.sqrt(252)
```

如果一个因子总是给高波动 ETF 高分，那么它的收益可能来自承担更高波动，而不是来自更好的排序能力。

这同样需要被诊断。

### 4. 中性化到底在做什么

中性化的核心代码是：

```python
neutral_factor = factor - X @ np.linalg.lstsq(X, factor, rcond=None)[0]
```

这句话看起来很数学，但思想可以很简单：

```text
把因子里能被 beta 和波动率解释的部分剔除掉；
留下不能被这些风险暴露解释的残差。
```

本章每个月做一次横截面回归：

```text
因子分数 = a + b1 * beta + b2 * volatility + 残差
```

中性化后的因子就是：

```text
残差
```

也就是：

```text
在扣掉 beta 和波动率影响之后，这个资产相对应该得多少分。
```

### 5. 用一个直觉例子理解

假设某个月 QQQ、XLK、XLY 动量分数很高。

但它们同时也有较高 beta 或较高波动率。

那么原始因子可能是在说：

```text
这些 ETF 最近涨得好。
```

中性化会追问：

```text
它们涨得好，是不是只是因为它们本来就更激进？
在同样 beta 和波动率水平下，它们还算不算强？
```

如果扣掉 beta 和波动率后，QQQ 仍然高分，那说明它的强可能不只是风险暴露。

如果扣掉后分数下降很多，说明原来的高分可能主要来自风险暴露。

### 6. 中性化不是一定让结果更好

这一点非常重要。

中性化不是魔法，不是说：

```text
中性化后一定更赚钱。
```

它只是帮你回答：

```text
这个因子的信息，是否独立于某些已知风险暴露？
```

如果 beta 或波动率本来就是你想赚的收益来源，中性化可能会把有效信息删掉。

如果 beta 或波动率是你不想要的噪声，中性化可能会让因子更干净。

所以中性化要先问目标：

```text
我想要纯 alpha？
还是我接受某种风险暴露？
```

### 7. 本章如何比较 raw 和 neutralized

本章比较两组因子：

```text
raw：原始动量因子
neutralized：扣掉 beta 和波动率后的动量残差因子
```

然后对它们分别做两件事：

```text
看 Rank IC
看 high_minus_low 多空研究组合
```

这和第 19、20 章是一脉相承的：

```text
先看排序能力；
再看高低分组相对收益。
```

### 8. 如何读本章结果

原始因子 `raw` 的结果是：

```text
final_equity = 0.8702
ann_return = -0.68%
max_drawdown = -47.11%
mean_rank_ic = -0.0171
icir = -0.0428
positive_ic_rate = 46.94%
```

中性化因子 `neutralized` 的结果是：

```text
final_equity = 1.1754
ann_return = 0.79%
max_drawdown = -34.45%
mean_rank_ic = 0.0068
icir = 0.0213
positive_ic_rate = 47.35%
```

中性化后，结果确实改善了一些：

```text
年化收益从 -0.68% 变成 0.79%
最大回撤从 -47.11% 变成 -34.45%
mean_rank_ic 从 -0.0171 变成 0.0068
```

但注意，改善后的结果仍然很弱：

```text
mean_rank_ic 仍然接近 0
ICIR 仍然接近 0
positive_ic_rate 仍然不到 50%
```

所以正确结论不是“中性化后因子变强了”，而是：

```text
中性化去掉了一些不想要的暴露，但剩下的信号仍然不够强。
```

### 9. exposure correlation 图在看什么

本章图里除了滚动 Rank IC，还看了：

```text
Average Exposure Correlation
```

它比较的是：

```text
因子分数和 beta 的相关性
因子分数和 volatility 的相关性
```

如果原始因子和 beta、波动率相关性较高，说明它可能混入了这些风险暴露。

如果中性化后相关性下降，说明中性化确实把这部分暴露压下去了。

这就是中性化作为诊断工具的价值。

### 10. 用 100W 账户怎么理解

如果用原始因子做高分组策略：

```text
每个月按 raw factor 排名；
买最高分组；
卖空最低分组或用它做研究对照。
```

如果用中性化因子：

```text
每个月先扣掉 beta 和波动率影响；
再按残差分数排名；
再选高分组和低分组。
```

比如 100W 账户只做多高分组时：

```text
raw 可能选到高 beta、高波动 ETF；
neutralized 可能选到在同等 beta 和波动率下仍然相对强的 ETF。
```

所以中性化改变的不是“买多少”的机制，而是：

```text
谁会被排到前面。
```

### 11. 中性化的风险

中性化也有风险。

第一，暴露估计本身有误差。比如 beta 用过去 252 天估计，未来 beta 可能变化。

第二，线性回归只是近似。真实市场关系不一定是线性的。

第三，资产数量太少。本章只有 15 个 ETF，每个月做横截面回归，样本不大。

第四，中性化可能删掉有效信息。如果高 beta 资产本来就是某段市场里的真实收益来源，你把 beta 中性掉，收益也可能被削弱。

### 12. 本章过关标准

你能讲清楚下面四句话，第 21 章就算过关：

```text
中性化是把因子中可由 beta、波动率解释的部分剔除。
中性化后的分数是残差分数，不是原始动量分数。
中性化是诊断工具，不保证提高收益。
本章结果说明中性化有改善，但改善后的因子仍然很弱。
```
