## 讲解

- 泄露特征直接包含未来收益，因此测试表现会异常漂亮。
- 这种漂亮结果不能交易，因为实盘当下拿不到未来收益。
- 机器学习量化第一原则：任何标准化、筛选和训练都只能使用过去数据。

## 详细讲解

### 1. 第 26 章为什么特别重要

第 25 章我们已经开始用机器学习模型生成交易信号。

但机器学习量化有一个非常危险的特点：

```text
模型越复杂，越容易把错误藏起来。
```

普通规则策略错了，你通常还能看出来：

```text
是不是均线写错；
是不是信号滞后；
是不是成本漏算。
```

机器学习模型错了，往往会表现成：

```text
回测曲线极其漂亮；
准确率极高；
回撤极小；
看起来像发现了圣杯。
```

但这种结果最常见的原因不是模型厉害，而是数据泄露。

所以第 26 章要训练你的第一反应：

```text
机器学习结果太好时，先怀疑泄露。
```

### 2. 什么是数据泄露

数据泄露就是：

```text
模型在训练或预测时，看到了当时本来不可能知道的信息。
```

量化交易里，最常见的泄露是未来数据进入了当前信号。

比如今天晚上你要决定明天是否买 SPY。你当时能知道的是：

```text
今天及以前的价格；
今天及以前的成交量；
已经发布的公开信息。
```

你不能知道：

```text
明天收益；
明天最高价；
明天最低价；
未来 20 天波动率；
未来财报结果；
未来指数成分变化。
```

如果这些未来信息混进特征，模型当然会表现得非常强。

但那不是预测未来，而是偷看答案。

### 3. 本章 proper model 是什么

本章有两个模型。

第一个是正常模型：

```text
proper_time_split_model
```

它只使用第 25 章那些当时可获得的特征：

```text
ret_5
ret_20
ret_63
vol_20
volume_z
```

训练方式也遵守时间顺序：

```python
train = data.iloc[:train_end]
test = data.iloc[test_start:]
```

意思是：

```text
前面的历史数据用来训练；
后面的未来区间只用来测试。
```

这个模型的测试结果是：

```text
final_equity = 2.3072
ann_return = 12.63%
ann_vol = 16.42%
max_drawdown = -22.24%
sharpe = 0.7688
direction_accuracy = 50.68%
```

这个表现不神奇，但比较真实。它像一个正常策略该有的样子：

```text
能赚钱，但有回撤；
准确率只比随机略高；
风险收益需要继续检验。
```

### 4. 本章 leaky model 是什么

第二个模型是故意做错的泄露模型：

```text
leaky_future_return_feature
```

它在特征里加入了：

```python
dataset["leaky_future_return"] = dataset["future_1d_return"]
```

这相当于直接把明天收益放进今天的输入。

换成考试类比就是：

```text
题目里夹着答案。
```

模型当然可以学得非常好，因为它不是真的预测未来，而是在读取未来。

### 5. 为什么泄露结果这么离谱

泄露模型的结果是：

```text
final_equity = 1838
ann_return = 191.22%
ann_vol = 12.12%
max_drawdown = -0.07%
sharpe = 15.7787
direction_accuracy = 99.94%
```

这个结果看起来像财富自由机器，但实际上完全没有交易价值。

因为实盘时你不可能在今天知道明天收益。

只要你看到类似结果，就应该马上警觉：

```text
高收益；
低回撤；
接近完美准确率；
长期稳定到不符合常识。
```

这通常不是 alpha，而是 bug、泄露或假设错误。

### 6. 机器学习为什么特别容易泄露

规则策略通常只有几个变量，容易检查。

机器学习策略会有很多步骤：

```text
特征构造；
标签构造；
标准化；
特征筛选；
模型训练；
参数搜索；
测试集评估；
信号转仓位。
```

每一步都可能不小心把未来放进去。

例如：

```text
用全样本均值标准化；
用全样本挑选最有效特征；
用测试集反复调参数；
把未来收益 shift 方向写反；
用未来成分股名单回测过去；
用已经修正后的财报数据模拟发布前交易。
```

这些错误不会让代码报错，但会让曲线变漂亮。

这就是机器学习量化最危险的地方。

### 7. 标准化为什么也会泄露

很多人只注意到不能把未来收益放进特征，却忽略标准化也可能泄露。

错误做法是：

```text
先用全样本计算均值和标准差；
再切分训练集和测试集。
```

这样测试集的信息已经进入了均值和标准差。

正确做法是：

```text
只用训练集计算均值和标准差；
再用这组参数处理测试集。
```

这和实盘一致：

```text
你只能用过去数据定义处理规则；
不能用未来数据反过来修正今天的输入。
```

### 8. 验证纪律是什么

验证纪律不是某一个函数，而是一组底线：

```text
训练数据必须早于测试数据；
特征必须在交易时点之前可获得；
标签只能用于训练和评价，不能进入预测输入；
预处理参数只能从训练集学习；
调参不能反复偷看最终测试集；
所有结果必须和简单基准比较。
```

机器学习量化真正要防的不是模型太弱，而是你以为模型很强。

一个正确但普通的模型，比一个漂亮但泄露的模型有价值得多。

### 9. 如何用 100W 账户理解本章

如果你有 100W，proper model 的逻辑是：

```text
模型预测明天收益为正 -> 买入 SPY；
模型预测不为正 -> 持现金。
```

它最终把测试期净值做到 2.3072，可以粗略理解成：

```text
测试期 100W 变成约 230.72W。
```

但中间最大回撤是 -22.24%，意味着账户可能从阶段高点跌掉约 22%。

leaky model 的 1838 倍净值不能这样理解，因为它实盘不可执行。

它不是：

```text
100W 变成 18.38 亿。
```

而是：

```text
代码把未来答案放进了模型，所以结果作废。
```

### 10. 本章过关标准

你能讲清楚下面四句话，第 26 章就算过关：

```text
数据泄露是模型看到了当时不可能知道的信息。
机器学习结果过分漂亮时，第一反应应该是检查泄露。
训练、标准化、特征筛选和调参都必须只使用过去数据。
泄露模型的收益曲线没有交易价值，哪怕指标看起来完美。
```
