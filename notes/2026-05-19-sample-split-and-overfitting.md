# Sample Split and Overfitting

日期：2026-05-19

## 本课目标

理解三个概念：

- 样本内：用来研究和调参的数据
- 验证集：用来比较少数候选参数的数据
- 样本外：最终检验区间，不能反复调参

以及一个核心纪律：

```text
不要把历史最优参数当成可靠参数。
```

## 为什么时间序列不能随机切分

股票数据是按时间发生的。真实交易中，未来数据不可能提前知道。

因此不能像普通机器学习那样随机打乱数据，而应该按时间切：

```text
过去 -> 现在 -> 未来
```

本课使用：

```text
训练区间：2000-01-01 到 2014-12-31
验证区间：2015-01-01 到 2019-12-31
测试区间：2020-01-01 到 2026-05-18
```

## 三个区间分别做什么

训练区间：

- 想策略
- 试参数
- 排除明显无效的规则

验证区间：

- 比较少数候选参数
- 判断参数是否有基本稳定性
- 避免只看训练区间

测试区间：

- 最终检验
- 不能反复调整
- 一旦用来改参数，就被污染了

## 参数敏感性

如果某个参数特别好，但旁边参数都很差，说明它可能只是刚好贴合历史噪声。

例如：

```text
20/100 很好
20/90 很差
20/110 很差
30/100 很差
```

这种参数不可靠。

更稳健的情况是：

```text
10/200 可以
20/200 可以
50/200 也可以
```

这说明策略可能不是依赖某个神奇数字，而是捕捉了较长期趋势。

## 如何解读

这个结果没有显示明显崩溃。训练区选出来的 `10/200` 在验证集和测试集仍然能跑。

但这不代表策略已经可靠，因为：

- 参数网格很小
- 只测试了 SPY
- 只测试了一个策略族
- 没有做交易成本压力测试
- 没有做 walk-forward
- 测试区包含 2020 之后的大行情，可能环境特殊

## 关键纪律

```text
样本外表现好，只能说明策略通过了一次考试。
样本外表现差，说明策略需要重新审视。
样本外被你反复拿来调参，就不再是样本外。
```
